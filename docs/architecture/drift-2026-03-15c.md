## Added

### Components (new nodes)
- `CONTEXT_BROKER` ‚Äî `Platform.Agents.ContextBroker` GenServer; manages parent‚Üíchild context sharing and child‚Üíparent delta promotion. Supervised under `Platform.Application` after the agent registry/runtime tree.
- `CONTEXT_SCOPE` ‚Äî `Platform.Agents.ContextScope` struct; declarative inheritance rules (`full | memory_only | config_only | custom`), max_depth enforcement, include/exclude key filtering.
- `CONTEXT_DELTA` ‚Äî `Platform.Agents.ContextDelta` struct; child‚Üíparent promotion payload (additions, removals, memory_updates, promote flag).
- `CONTEXT_SHARE` ‚Äî `Platform.Agents.ContextShare` Ecto schema; persists provenance records in `agent_context_shares` for every share and promotion event.
- `CONTEXT_BROKER_INT` ‚Äî External integration map node for ContextBroker showing its DB footprint and coordination role.

## Summary

This is a **structural drift** event. 4 new components and 10 new edges added to the agent runtime subgraph. The ContextBroker is the key coordination GenServer for the multi-agent context inheritance protocol. No new DB tables were added (agent_context_shares was created in the T3 schema migration), but the table is now actively used via ContextShare inserts on every share/promote call.
